# WAF Log Monitor (Nginx, ModSecurity, Splunk)

## Overview
This project implements a Web Application Firewall (WAF) system to detect and block common web attacks such as SQL Injection and Cross-Site Scripting (XSS). It also integrates centralized log monitoring using Splunk for real-time analysis.

## Objectives
- Protect web applications from common attacks  
- Monitor and analyze web traffic logs  
- Detect suspicious requests and attack patterns  

## Architecture
- Nginx (OpenResty) as reverse proxy  
- ModSecurity for web application firewall rules  
- Splunk for centralized log collection and monitoring  

## Features
- Detect and block SQL Injection and XSS attacks  
- Custom security rules using ModSecurity and Lua  
- Real-time log monitoring and analysis with Splunk  
- Attack simulation for testing detection capabilities  

## Technologies
- Nginx / OpenResty  
- ModSecurity  
- Lua  
- Splunk  

## Results
- Successfully detected and blocked web-based attacks  
- Centralized logs for monitoring and analysis  
- Improved understanding of web security and incident detection
